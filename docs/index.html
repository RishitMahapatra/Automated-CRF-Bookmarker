Background

What is a CRF & why does this tool exist?

Understanding the domain problem the tool solves.

In clinical trials, a Case Report Form (CRF) is a specialized questionnaire designed to capture protocol-required data for every study participant — recording patient demographics, medical history, vital signs, lab results, adverse events, and efficacy endpoints.

During data management and auditing, CRFs from multiple patient visits are bundled into massive, multi-page PDF compilations — often spanning hundreds or thousands of pages — with no internal navigation structure.

⚠️
The Problem: Clinical Data Managers, Monitors (CRAs), and Biostatisticians waste hours manually hunting for specific patient records in un-bookmarked PDFs. Missing a section during audit review can cause regulatory non-compliance.
✅
The Solution: CRF Auto-Bookmarker detects Folder: and Form: headers embedded in each page and constructs a two-level hierarchical Table of Contents directly inside the PDF — reducing navigation time from hours to seconds.

Algorithm

How it works

The engine reads each page's text via PyMuPDF (fitz) and builds a collapsed Table of Contents.

1

Open PDF in memory

The engine reads the input file as bytes and opens it with fitz.open(stream=bytes, filetype="pdf") — no temp file written to disk.

2

Scan the top region of each page

For every page, text blocks with a top-Y coordinate within the configurable clip height (default 140 points from the top) are extracted using page.get_text("dict").

3

Extract Folder and Form labels

Lines containing Folder: and Form: tokens are identified. The text after the colon becomes the bookmark label.

# Example page header content
Folder: Screening Visit (Week 0)
Form:   Adverse Events
4

Build the Table of Contents

A TOC list is assembled where each entry is [level, title, page_number]. Level 1 entries are Folders; Level 2 entries are Forms nested within them. New entries are added only when the folder or form name changes.

toc = [
  [1, "Screening Visit (Week 0)", 1],
  [2, "Adverse Events",           1],
  [2, "Vital Signs",              14],
  [1, "Week 4 Follow-up",         28],
  ...
]
5

Write bookmarks back into PDF

doc.set_toc(toc, collapse=1) embeds the TOC with bookmarks collapsed by default. The output is serialised with garbage=4, deflate=True for optimal file size.

User Manual

Using the Application

Step-by-step walkthrough for generating bookmarks with the desktop GUI.

1

Select your CRF PDF

Click the Browse button next to the Input PDF field. A file picker dialog will open. Navigate to your un-bookmarked CRF PDF and click Open. The file path will appear in the input field.

📋
The tool works with any text-based CRF PDF that contains visible Folder: and Form: labels in the page header area. Image-only (scanned) PDFs require OCR pre-processing — see Troubleshooting.
2

Adjust header scan height (optional)

The Header scan height slider controls how far down from the top of each page the engine searches for Folder: and Form: labels.

SettingValueWhen to use
Default140 ptsWorks for most standard CRF templates
Lower100–130 ptsHeaders are positioned very close to the top
Higher160–250 ptsHeaders are placed further down or span multiple lines

Drag the slider left/right. The current value in points is displayed immediately to the right of the slider.

3

Generate Bookmarks

Click the 🚀 Generate Bookmarks button. The engine processes your PDF in a background thread:

  • The progress bar fills as each page is processed.
  • A status label shows Processing page X of Y...
  • The button is disabled during processing to prevent double-clicks.

Developer Reference

Programmatic API

Use bookmarker.py directly in your own Python scripts or pipelines without launching the GUI.

process_crf_bytes(input_bytes: bytes, clip_height: float = 140, progress_callback=None) → BookmarkResult

Processes a CRF PDF entirely in memory and returns the bookmarked PDF as bytes. No temporary files are created.

  • input_bytes bytes — Raw bytes of the input PDF file.
  • clip_height float — Height from the top of each page (in points) to scan for headers. Default: 140.
  • progress_callback callable(current, total) — Optional function called with page progress. Use for custom progress tracking.

BookmarkResult dataclass

FieldTypeDescription
successboolWhether bookmarking succeeded
total_pagesintNumber of pages in the PDF
parent_countintNumber of Folder-level (L1) bookmarks
child_countintNumber of Form-level (L2) bookmarks
total_bookmarksintTotal bookmarks inserted
pages_skippedintPages where no header was found
unique_foldersList[str]Sorted list of unique Folder names found
unique_formsList[str]Sorted list of unique Form names found
output_bytesOptional[bytes]The bookmarked PDF as bytes (None on failure)
error_messageOptional[str]Human-readable error description (None on success)

Example usage

from bookmarker import process_crf_bytes

# Read input PDF
with open("ClinicalTrial_Batch01.pdf", "rb") as f:
    input_bytes = f.read()

# Process (increase clip_height if headers are lower on the page)
result = process_crf_bytes(input_bytes, clip_height=140)

if result.success:
    with open("ClinicalTrial_Batch01_Bookmarked.pdf", "wb") as f:
        f.write(result.output_bytes)
    print(f"✅ Created {result.parent_count} folders, {result.child_count} forms")
    print(f"   {result.total_bookmarks} total bookmarks across {result.total_pages} pages")
else:
    print(f"❌ Failed: {result.error_message}")
python

Batch processing example

import pathlib
from bookmarker import process_crf_bytes

input_dir  = pathlib.Path("./raw_crfs")
output_dir = pathlib.Path("./bookmarked_crfs")
output_dir.mkdir(exist_ok=True)

for pdf_path in input_dir.glob("*.pdf"):
    input_bytes = pdf_path.read_bytes()
    result      = process_crf_bytes(input_bytes)

    if result.success:
        out_path = output_dir / f"{pdf_path.stem}_Bookmarked.pdf"
        out_path.write_bytes(result.output_bytes)
        print(f"✅ {pdf_path.name} → {result.total_bookmarks} bookmarks")
    else:
        print(f"❌ {pdf_path.name}: {result.error_message}")

Help

Troubleshooting

Common issues and how to resolve them.

❌ Zero bookmarks generated

The engine couldn't find Folder: or Form: labels.

Fix: Try increasing the Header scan height slider to 180–250 pts. Verify your PDF actually contains these literal text labels in the page header.

❌ Failed to open PDF

The PDF file could not be parsed by PyMuPDF.

Fix: Check if the file is corrupted (open it in a PDF reader first). Re-export or re-download the PDF if needed.

❌ Scanned / image-only PDF

The PDF contains scanned images with no embedded text layer.

Fix: Run OCR first (e.g., Adobe Acrobat, Tesseract, or ABBYY FineReader) to embed a text layer, then process with this tool.

❌ ModuleNotFoundError: tkinter

Tk is not installed on your system (common on Linux).

Fix (Debian/Ubuntu): sudo apt-get install python3-tk
Fix (Fedora/RHEL): sudo dnf install python3-tkinter

❌ Some pages skipped

Cover pages, blank pages, or pages with a different layout won't have matching headers — this is expected.

Not a bug: Skipped pages are reported in the results console for transparency.

❌ Folders and Forms mixed up

The header text contains Folder: and Form: labels but in an unexpected position.

Fix: Slightly increase or decrease the Header scan height to fine-tune which text blocks are captured.
